How It Works

One alias. Every library.

Point your bundler's react import at what-react. Every React library in your dependency tree automatically runs on What's signals-based runtime ‚Äî no forks, no patches, no wrappers.

01
Alias in your config
Add a single resolve alias in Vite, webpack, or esbuild. react and react-dom now point to what-react.
02
Import as usual
Keep using useState, useEffect, createContext, and every React hook you know. Your existing code doesn't change.
03
Run on signals
Under the hood, hooks map to What's fine-grained reactivity. No virtual DOM diffing ‚Äî only the specific DOM nodes that depend on changed data update. 
// vite.config.js
export default {
  resolve: {
    alias: {
      'react':             'what-react',
      'react-dom':         'what-react/dom',
      'react-dom/client':  'what-react/dom',
      'react/jsx-runtime': 'what-react/jsx-runtime',
    }
  }
}

Architecture

Not a shim. A runtime.

The compat layer isn't a hack ‚Äî it's a complete implementation of React's public API on top of What's signals-based runtime. Every hook, every pattern, backed by fine-grained reactivity.

‚ö°

Fine-grained Signals

useState, useEffect, useMemo all map to signals. Only the DOM nodes that depend on changed data update ‚Äî no tree diffing.

üèó

Full Context Support

createContext, Provider, Consumer, and useContext all work. Nested providers, default values, render-prop consumers.

üß¨

Class Components

ES6 class components with lifecycle methods (componentDidMount, componentDidUpdate, componentWillUnmount) are automatically detected and wrapped.

üîÄ

Portal & Ref Support

createPortal renders into external DOM nodes. forwardRef, useImperativeHandle, and createRef work as expected.

üîÑ

React 18 APIs

useSyncExternalStore, useTransition, useDeferredValue, useId, Suspense, and lazy loading are all implemented.
