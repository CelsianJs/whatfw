Fine-grained reactivity without a virtual DOM. Write components with familiar patterns, ship minimal JavaScript. Built for performance from the ground up.

import { signal } from 'what-framework';

  const count = signal(0);

  return (

    </button>

Features

Built for simplicity and speed

What combines the best patterns from React, Svelte, and SolidJS into a lightweight package.

Fine-Grained Signals

Reactive primitives that track dependencies automatically. When a signal changes, only the DOM nodes that use it update. No component re-renders.

No Virtual DOM

Updates go directly to the DOM. No diffing algorithms, no reconciliation overhead. Just surgical updates where they're needed.

Tiny Bundle

The core runtime is under 4kB gzipped. Zero dependencies. Ships less JavaScript to your users.

Islands Architecture

Ship static HTML by default. Hydrate interactive components on idle, on visible, or on first interaction.

Familiar Patterns

JSX syntax, React-like hooks, component composition. Use patterns you already know with better performance characteristics.

SSR Built-In

Server-side rendering with streaming support. Render to string or stream HTML chunks for faster time-to-first-byte.

Examples

Code that reads clearly

Signals make state management straightforward. No magic, just reactive values.

Reactive State

Signals hold values that can change over time

import { signal, computed } from 'what-framework';

// Create reactive values
const firstName = signal('John');
const lastName = signal('Doe');

// Derived values update automatically
const fullName = computed(() =>
  `${firstName()} ${lastName()}`
);

// Update the source, derived updates too
firstName.set('Jane');
console.log(fullName()); // "Jane Doe"

Side Effects

Run code when dependencies change

import { signal, effect } from 'what-framework';

const query = signal('');

// Runs when query changes
effect(() => {
  console.log('Searching:', query());
});

// Effects can return cleanup functions
effect(() => {
  const id = setInterval(() => {
    console.log(query());
  }, 1000);
  return () => clearInterval(id);
});

Components

Functions that return elements

function TodoList() {
  const todos = signal([]);
  const input = signal('');

  const addTodo = () => {
    todos.set(t => [...t, input()]);
    input.set('');
  };

  return (
    <div>
      <input
        value={input}
        onInput={e => input.set(e.target.value)}
      />
      <button onClick={addTodo}>Add</button>
      <ul>
        {() => todos().map(todo => <li>{todo}</li>)}
      </ul>
    </div>
  );
}

Islands

Hydrate components when needed

// Static by default - no JS shipped
<Header />
<Footer />

// Hydrate when the element is visible
<Comments client:visible />

// Hydrate when browser is idle
<Analytics client:idle />

// Hydrate on first click/focus/hover
<Dropdown client:interaction />

// Hydrate immediately
<Counter client:load />

Get Started

Up and running in seconds

$ npx create-what my-app
  • 1 Run the create command to scaffold a new project
  • 2 Navigate to your project: cd my-app
  • 3 Install dependencies: npm install
  • 4 Start the dev server: npm run dev

Or add to an existing project

Install the package and start using signals

# Install the framework
npm install what-framework

# In your code
import { signal, effect } from 'what-framework';
import { h, mount } from 'what-framework';

function App() {
  const message = signal('Hello, World!');

  return h('h1', {}, () => message());
}

mount(h(App), '#app');
