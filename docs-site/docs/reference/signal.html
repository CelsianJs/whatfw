signal()

Create a reactive value that can be read and updated.

const count = signal(initialValue)

Parameters

ParameterTypeDescription
initialValue T The initial value for the signal

Returns

A signal object with the following methods:

MethodDescription
signal() Read the current value (creates subscription in reactive contexts)
signal.set(value) Set a new value
signal.set(fn) Update based on current value
signal.peek() Read value without creating a subscription

Usage

Basic Usage

import { signal } from 'what-framework';

const count = signal(0);

// Read
console.log(count());  // 0

// Update
count.set(5);
console.log(count());  // 5

// Update with function
count.set(c => c + 1);
console.log(count());  // 6

In Components

function Counter() {
  const count = signal(0);

  return (
    <button onClick={() => count.set(c => c + 1)}>
      Count: {count()}
    </button>
  );
}

With Objects

const user = signal({ name: 'Alice', age: 25 });

// Update - must create new reference
user.set(u => ({ ...u, age: 26 }));

With Arrays

const items = signal([1, 2, 3]);

// Add item
items.set(arr => [...arr, 4]);

// Remove item
items.set(arr => arr.filter(x => x !== 2));

Reading Without Subscribing

import { signal, effect } from 'what-framework';

const count = signal(0);
const multiplier = signal(2);

effect(() => {
  // Uses count but doesn't subscribe to multiplier
  const result = count() * multiplier.peek();
  console.log(result);
});

Notes

  • Signals should not be mutated directly. Always use .set() to update.
  • In JSX, read signals with (): {count()}
  • Use peek() when you need the value but don't want to create a subscription
  • For derived values that depend on other signals, use computed()
